# 武当内容创作系统 - 开发规范

## 分支管理规范

### 主分支
- **唯一主分支**: `main`
- **禁止**: 创建其他开发分支（如master、dev等）
- **所有开发**: 必须在main分支上直接进行

### 提交规范
```
版本号: 功能描述

- 新增/修改的文件列表
- 功能说明
```

示例：
```
v3.5: 数据中心增强

- 新增数据导出功能
- 优化图表展示
- 修复统计计算bug
```

## 开发流程

1. **每次开发前**: `git pull origin main` 拉取最新代码
2. **开发完成后**: `git add .` + `git commit -m "版本: 描述"`
3. **推送到GitHub**: `git push origin main`
4. **验证**: 检查GitHub上main分支是否更新

## 承诺

**今后所有开发迭代都将严格遵循以下原则：**

1. ✅ **单分支开发**: 只在main分支上开发
2. ✅ **完整提交**: 每次提交包含所有相关文件
3. ✅ **版本清晰**: 每个版本功能完整、可追溯
4. ✅ **及时同步**: 开发完成后立即推送到GitHub
